## ADDED Requirements

### Requirement: Scheduler log path is configurable
The system SHALL allow the log file path for cron job output redirection to be configured via `scheduler.logPath` in `config.yaml`. The default value SHALL be `/app/cron-madz.log`.

#### Scenario: Default log path is used when not configured
- **WHEN** `scheduler.logPath` is not set in config.yaml
- **THEN** the scheduler uses `/app/cron-madz.log` as the log file path

#### Scenario: Custom log path is respected
- **WHEN** `scheduler.logPath` is set to `/var/log/custom-cron.log` in config.yaml
- **THEN** the scheduler uses `/var/log/custom-cron.log` as the log file path

### Requirement: Crontab entries use configurable log path
The system SHALL use the configured `scheduler.logPath` value in all crontab entries generated by `prepareCrontabCommand()`.

#### Scenario: prepareCrontabCommand uses configurable path
- **WHEN** `prepareCrontabCommand()` is called with a command string
- **THEN** the returned string appends `>> /app/cron-madz.log 2>&1` (or the configured path)

#### Scenario: All crontab methods use the configurable path
- **WHEN** `Cron.add()`, `Cron.install()`, or `Cron.sync()` generates crontab entries
- **THEN** all entries use the configured log path from `scheduler.logPath`

### Requirement: Writable path validation
The system SHALL warn when the configured log path is not writable at schedule creation time.

#### Scenario: Warning logged for unwritable path
- **WHEN** the configured log path is not writable
- **THEN** a warning is logged but the crontab entry is still created
