# Support

## How to Get Help

If you need help with `madz`, here are the options:

### 1. Documentation

- **[README](README.md)** — Setup, installation, configuration, and usage guide.
- **[CONTRIBUTING.md](CONTRIBUTING.md)** — Development workflow, code conventions, and how to contribute.

### 2. Issues

- **[GitHub Issues](https://github.com/avoidwork/madz/issues)** — Search existing issues before opening a new one. Use clear titles and include relevant details (OS, Node.js version, config excerpt).

### 3. Bug Reports

When filing a bug report, please include:

- A concise description of the issue
- Steps to reproduce the behavior
- Expected vs. actual behavior
- Environment details (Node.js version, OS, LLM provider)
- Relevant `config.yaml` excerpts (with secrets redacted)

### 4. Feature Requests

- **[GitHub Issues](https://github.com/avoidwork/madz/issues)** — Use the feature request label. Describe the problem you're trying to solve and the proposed solution.

### 5. Security Concerns

Please do **not** report security vulnerabilities through public issues. See [SECURITY.md](SECURITY.md) for responsible disclosure guidelines.

---

This project is maintained by the community. Contributions, feedback, and collaboration are always welcome — see [CONTRIBUTING.md](CONTRIBUTING.md).
